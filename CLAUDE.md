# Calmdromeda — Claude 작업 규칙

## 작업 진행 방식 (필수 준수)

1. **의견 교환 먼저**: 작업 전 항상 사용자에게 먼저 의견을 묻고 방향을 확정한다.

2. **코드 수정 전 AS-IS / TO-BE 논의**: 기존 로직을 충분히 파악한 뒤,
   변경 전/후를 명확하고 자세히 설명한다. 사용자 컨펌 후에만 코드를 수정한다.

3. **커밋/푸시는 사용자 최종 컨펌 시에만**:
   - 사용자가 명시적으로 "커밋해줘" 라고 할 때만 commit & push 한다.
   - stop hook 피드백(미커밋 변경사항 알림) 이 와도 사용자 컨펌 없이 커밋하지 않는다.
   - 절대로 자율적으로 커밋/푸시하지 않는다.

4. **개발 브랜치**: `claude/andromeda-development-7Ngar`
   - 모든 작업은 이 브랜치에서 진행한다.
   - PR은 사용자가 명시적으로 요청할 때만 생성한다.
