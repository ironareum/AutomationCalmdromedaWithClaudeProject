4. 실 사용 파일만 output 적재  → 디스크 사용량 최소화
    (as-is):
        collect() → 파일 수집 → 바로 _used/ 이동 (사용 전에 이동!)
        output/sounds/ → 다운받은 것 전부 (5개 중 3개만 써도 5개 다 남음)

    (to-be):
      collect() → 파일 수집 (이동 X)
      produce() 완료 → 실제 사용한 파일만 _used/ 이동
      output/sounds/ → 실제 사용한 파일만 남기고 나머지 삭제
      output/videos/ → 실제 사용한 파일만 남기고 나머지 삭제

    Step 1: collect() — 파일 수집만, 이동 없음
    Step 2: collect() — 영상 수집만
    Step 3: produce() — FFmpeg으로 실제 사용
             ↓ 완료 후 _cleanup_assets() 호출
             ├─ 로컬 음원(assets/sounds/) → _used/{session_id}/ 이동
             ├─ output/sounds/ 미사용 파일 → 삭제
             └─ output/videos/ 미사용 파일 → 삭제
    Step 4: 썸네일 생성 (video_files[0] 이미 정리됐으면 안되니 썸네일이 먼저)

    판단 기준:
        sound_files, video_files = produce()에 실제로 넘긴 리스트 → 이것만 "사용한 파일"로 간주
        로컬 음원 여부: LOCAL_SOUNDS_DIR in sound_path.parents 로 판별 (API 다운로드 파일은 이동 안 함)

5. output/{session_id}/pipeline.log 로그 파일 자동 생성

6. used_asset.json 포맷 변경
    e.g.)

    session이 key라서 output 폴더명과 1:1 매칭되고, sounds/videos 파일명만 지우면 해당 소스 재사용 가능.
    **원하는 대로 쓰는 방법:**
    # 소스 재사용하고 싶으면
    → used_assets.json에서 해당 session 블록 전체 삭제
    → pipeline.py 재실행

    # 특정 음원만 재사용
    → 해당 session의 "sounds" 배열에서 파일명 삭제
    → assets/sounds/_used/{session_id}/ 에서 해당 파일을 원래 카테고리 폴더로 이동
    → pipeline.py 재실행
